# 客語羅馬拼音索引施工計畫

## 目標

- [ ] 讓新專案 `autocomplete` 支援客語羅馬字查詢（行為對齊舊站）。
- [ ] 採用「預建索引 + R2 靜態查詢」架構，避免查詢時做重運算。
- [ ] 以 Node/TS 腳本延伸既有 `scripts/build-pinyin-lookup.mjs`，不回退 Perl 維運。
- [ ] 第一版支援客語兩套拼音類型：`TH`、`PFS`。

## 舊站行為確認（已知事實）

- [x] 舊站前端 `pinyin_lookup` 會對 `LANG=h` 請求 `lookup/pinyin/h/{type}/{term}.json`。
- [x] 舊站拼音偏好來自 `localStorage.pinyin_h`，查詢型別可切換。
- [x] 舊站 `build-pinyin-lookup.pl` 允許以 `h` 語別建索引（資料源 `dict-hakka.json`）。
- [x] 已補查 `build-pinyin-lookup.pl`：客語索引來源欄位是 `heteronyms[].pinyin`，token 規則為 `NFD -> 去 combining mark -> ɑ→a -> 依 [a-z]+ 切詞`，腳本本身未直接產生 `TH/PFS` 型別檔。
- [x] 已確認舊站有 `/lookup/pinyin/h/TH/ah.json`、`/lookup/pinyin/h/PFS/ah.json` 路由。

> note:
> - 目前可確定舊站使用 `lookup/pinyin/h/{type}/{term}.json` 靜態檔查詢。
> - 依目前舊碼，客語索引建置可確認只有 `pinyin` 欄位與通用 token 正規化；`TH/PFS` 檔名型別的映射/後處理不在此檔，需於新專案明確定義。

## Phase 0：規格確認（先做）

- [x] 開發分支：`new-feature-pinyin-lookup`。
- [x] 客語拼音類型定稿：第一版固定支援 `TH`、`PFS`，預設取 `localStorage.pinyin_h`（無值時預設 `TH`）。
- [x] 輸入判斷規則：以英文字輸入查詢（非中文字）時，判定為「羅馬字查詢」，放寬可用客語專用符號。
- [x] 建議 regex 草案（接受拉丁擴展字母、tone 數字/上標、空白、連字號、撇號）：`/^(?=.*[A-Za-z\u00C0-\u024F\u1E00-\u1EFF])[A-Za-z\u00C0-\u024F\u1E00-\u1EFF\u0300-\u036F0-9¹²³⁴⁵ⁿ\s\-']+$/u`。
- [x] fallback 規則：羅馬字查無候選詞時，不回退一般索引，也不進入 404 字圖頁。
- [x] 排序規則：不做「羅馬索引 + 一般索引」合併排序，兩種查詢路徑互斥。

> note:
> - 你目前需求是「客語可使用一般索引 + 羅馬字搜尋索引」，需先定義合併策略（交集、聯集、排序權重）。
> - 若維持與台語相同互斥路徑，實作最單純；若要雙路徑合併，需先設計去重與排序規則。
